## Fase 1 — Supabase novo (schema enxuto)

Criar projeto Supabase novo. Rodar `sql/001_nucleo.sql` no SQL Editor.

### Tabelas — só estas 4 no núcleo

**1. `professionals`**
- `id` UUID PK
- `slug` TEXT UNIQUE NOT NULL (URL pública: `/p/?slug=`)
- `name` TEXT NOT NULL
- `specialty` TEXT (opcional no núcleo — pode ser NULL)
- `avatar_url` TEXT
- `whatsapp` TEXT (para Página de Reputação Semana 3 — pode NULL agora)
- `average_rating` NUMERIC(3,2) DEFAULT 0
- `total_reviews` INTEGER DEFAULT 0
- `created_at`, `updated_at` TIMESTAMPTZ

**NÃO incluir:** igv_score, salary_expectation, tags arrays, seeking_work, gallery, etc.

**2. `qr_sessions`** (sessão QR — token único por atendimento)
- `id` UUID PK
- `professional_id` UUID FK → professionals
- `token` TEXT UNIQUE NOT NULL (UUID v4 ou nanoid — imprevisível)
- `expires_at` TIMESTAMPTZ NOT NULL (ex.: NOW() + 2 horas — configurável)
- `used_at` TIMESTAMPTZ NULL (preencher ao submeter avaliação — token single-use)
- `created_at` TIMESTAMPTZ

**Regras:**
- Token expirado → bloqueia avaliação
- Token já usado (`used_at` NOT NULL) → bloqueia avaliação
- Sem bypass por auth, query string alternativa ou endpoint paralelo

**3. `reviews`**
- `id` UUID PK
- `professional_id` UUID FK NOT NULL
- `qr_session_id` UUID FK NOT NULL UNIQUE (1 avaliação por sessão QR)
- `rating` INTEGER CHECK (1–5)
- `comment` TEXT
- `is_verified` BOOLEAN DEFAULT TRUE (sempre true neste MVP — só entra via QR)
- `created_at` TIMESTAMPTZ

**NÃO incluir:** user_id obrigatório, review_type, establishment_id, prof_link_snapshot, hiring — Semana 3+.

**4. `establishments`** — **NÃO criar nesta task.** Dashboard e cadastro são Semana 3–4.

### Função + trigger
- `recalc_professional_rating(professional_id)` — recalcula `average_rating` e `total_reviews` após INSERT em `reviews`
- Trigger `AFTER INSERT ON reviews`

### RLS (MVP pragmático)
- `professionals`: SELECT público (anon) para perfil; INSERT/UPDATE só authenticated ou service_role (seed manual ok no dev)
- `qr_sessions`: INSERT authenticated (profissional gera); SELECT anon só por token exato via RPC ou policy restrita
- `reviews`: INSERT anon **somente** se `qr_session` válido — preferir **Edge Function** ou RPC `submit_qr_review(token, rating, comment)` que valida expiração/uso internamente

**Importante:** não deixar INSERT direto em `reviews` sem validação de token. Fraude = produto morto.

## Fase 2 — Rotas e fluxo (3 telas)

### Mapa de URLs

| Rota | Arquivo | Função |
|------|---------|--------|
| `/qr/?token=XXX` | `qr/index.html` | Valida token → redirect para avaliar ou erro |
| `/avaliar/?token=XXX` | `avaliar/index.html` | Form nota (1–5) + comentário |
| `/p/?slug=joao-barbeiro` | `p/index.html` | Perfil público: nota média + histórico |

### Fluxo feliz (end-to-end)

```
1. Profissional (dev: botão/seed) gera qr_session → recebe URL com token
2. Cliente abre URL no celular
3. Sistema valida: token existe, não expirou, não foi usado
4. Cliente envia nota + comentário
5. review persiste; qr_session.used_at = NOW()
6. Perfil público /p/?slug=... mostra nova avaliação + nota atualizada
```

### Fluxo de erro (obrigatório — UI clara)

| Caso | Comportamento |
|------|---------------|
| Token inexistente | Tela: "QR inválido" — sem formulário |
| Token expirado | Tela: "Este QR expirou. Peça um novo ao profissional." |
| Token já usado | Tela: "Esta avaliação já foi registrada." |
| Submit duplo | Idempotente — segunda tentativa = mensagem já usado |

### Tela de avaliação (`avaliar/index.html`)
- Nota: 5 estrelas (toque grande — mobile first)
- Comentário: textarea opcional, máx. 500 chars
- Botão único: "Enviar avaliação"
- Sem login, sem cadastro, sem campos extras
- DNA Apple: espaçamento generoso, tipografia limpa, feedback de sucesso com link pro perfil do profissional

### Perfil público (`p/index.html`)
- Nome + specialty (se houver)
- Nota média grande + contagem ("4.8 · 12 avaliações")
- Lista cronológica: estrelas + comentário + data relativa
- Empty state: "Ainda sem avaliações verificadas"
- **Sem:** marketplace, match, botão contratar, IGV, drawer

### Gerador de QR (dev/MVP interno)
Para Semana 1–2, pode ser página mínima `dev/gerar-qr.html` ou seed script:
- Seleciona profissional (dropdown do seed)
- Cria `qr_session` com token + expires_at
- Mostra QR code (lib leve: ex. `qrcode.js` via CDN) + URL copiável
- **Não** precisa ser bonito — precisa funcionar em produção
